    Class AuthService

    Servicio principal de autenticación que maneja login, registro y validación de usuarios.

    Description

    Proporciona métodos para:

    • Validar credenciales de usuario (email/password)
    • Generar tokens JWT para autenticación
    • Registrar nuevos usuarios con autenticación automática
    • Validar tokens JWT en requests subsecuentes

    login

    • login(loginInput: LoginInput): Promise<AuthResponse>

      Autentica un usuario y genera un token JWT de acceso.

      Parameters

      • loginInput: LoginInput

        Objeto con email y password del usuario

      Returns Promise<AuthResponse>

      Respuesta con token de acceso y datos completos del usuario

      Description

      Realiza el proceso completo de login:

      1. Valida las credenciales del usuario
      2. Actualiza la fecha de último login
      3. Genera un token JWT con información del usuario y roles
      4. Retorna el token junto con los datos del usuario

      Throws

      Cuando las credenciales son incorrectas o el usuario está inactivo

      Registra un nuevo usuario en el sistema y lo autentica automáticamente.

      Parameters

      • registerInput: CreateUserInput

        Datos del nuevo usuario (email, password, full_name, etc.)

      Returns Promise<AuthResponse>

      Respuesta con token de acceso y datos del usuario registrado

      Description

      Crea un nuevo usuario en la base de datos y genera inmediatamente un token de acceso para el usuario recién registrado. El rol se asigna automáticamente según la lógica definida en el UsersService.

      Throws

      Si hay problemas con la creación del usuario

    • validateJwt(payload: any): Promise<any>

      Valida un token JWT y retorna la información del usuario.

      Parameters

      • payload: any

        Payload decodificado del token JWT que contiene email, sub (id), rol, etc.

      Returns Promise<any>

      Información básica del usuario para inyectar en el contexto de la request

      Description

      Verifica que el payload del JWT sea válido y que el usuario exista y esté activo en la base de datos. Este método es utilizado por la estrategia JWT de Passport para validar tokens en cada request.

      Throws

      Cuando el usuario no existe o está desactivado
